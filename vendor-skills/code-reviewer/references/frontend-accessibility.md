# Frontend: Accessibility (a11y)

**Impact: MEDIUM** | **Category: frontend** | **Tags:** a11y, wcag, semantics

Accessible UI is a quality and (for public-sector-adjacent products) often a compliance concern.

## Checklist
- [ ] Semantic HTML; images have alt text; form inputs have associated labels.
- [ ] Keyboard navigable; visible focus states; no keyboard traps.
- [ ] Colour contrast meets WCAG AA; colour is not the only signal.
- [ ] ARIA used correctly (and only where semantics are insufficient).
- [ ] Dynamic updates announced (aria-live) where relevant.
